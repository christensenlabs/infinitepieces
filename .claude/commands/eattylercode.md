into this repository. It is in the `tylers-code` folder. It contains some mocked data, some components that are duplicates
of the components in this repo, and some new components, that Tyler wants integrated into this application.

Set /effort to maximum, and try and integrate his code into this repository. Read all of the code in that folder and carefully
integrate it into this application, with the following rules:

- Ensure that data is not hardcoded into components. Do your best to asses when data is hardcoded, and instead
acquire the data by making an API call. The API call should be mocked out, to be integrated with a backend environment
later.

- Ensure that components and views and styles are composed with the same rules as the rest of this application.
